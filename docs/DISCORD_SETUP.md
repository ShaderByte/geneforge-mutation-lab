# ShaderByte Discord Setup

This guide creates a Discord server for GeneForge Mutation Lab support, chat, and build discussion without relying on third-party bots.

## Server Name

`ShaderByte Community`

## Public Invite

https://discord.gg/RJxXB8Fsp

## Safety Settings

Use Discord's built-in safety tools:

- Enable Community features.
- Set Verification Level to `Highest`.
- Enable Rules Screening.
- Enable AutoMod.
- Do not add third-party bots.

`Highest` verification requires members to meet Discord's strongest built-in participation checks, including a verified phone number. This does not guarantee every user is human, but it is the strongest no-bot baseline Discord provides without external bot verification.

## Roles

Create these roles:

- `Owner`
- `Moderator`
- `Verified Member`
- `Support Helper`

Keep `@everyone` limited:

- Can view public channels after passing Discord's verification and Rules Screening.
- Cannot manage channels, roles, webhooks, invites, or messages.
- Cannot mention everyone/here.

## Suggested Rules

1. Be respectful. No harassment, hate speech, threats, or personal attacks.
2. Keep discussion related to GeneForge, ARK stat planning, builds, support, or ShaderByte projects.
3. Do not post malware, cracked software, suspicious files, or fake download links.
4. Do not impersonate ShaderByte, moderators, Studio Wildcard, Snail Games, or other creators.
5. Use support channels for bugs and app issues.
6. No spam, raids, scams, or automated accounts.
7. Do not share personal information.
8. The official app download link is the GitHub repository only unless ShaderByte announces otherwise.

## AutoMod Suggestions

Enable built-in AutoMod rules for:

- Common spam
- Mention spam
- Suspicious links
- Harmful or explicit words
- Blocked words related to scams, token grabbers, and fake verification

Recommended blocked phrases:

- free nitro
- steam gift
- crypto giveaway
- verify with bot
- paste this in powershell
- run this script
- token
- webhook spam

## Invite Link Settings

When you create the public invite:

- Set it to never expire.
- Limit who can create new invites to moderators.
- Post the invite only in trusted places, starting with the GitHub README.
